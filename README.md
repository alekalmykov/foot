# Football Poll Telegram Bot

Простой Telegram-бот на Python для общих футбольных опросов и записи участников.

Рекомендуемый способ запуска на Linux: через Docker и `docker compose`.

## Возможности

- `/start` — приветствие и список команд
- `/newpoll` — создать новый опрос "Кто идёт на футбол?"
- `/current` — показать текущий активный опрос в чате
- `/closepoll` — закрыть текущий опрос в чате
- Inline-кнопка `✅ Иду` работает как toggle
- Общий shared-state: изменения видны всем во всех связанных сообщениях одного опроса
- Хранение данных в SQLite
- Еженедельное автоматическое создание нового опроса через APScheduler

## Структура проекта

- `main.py` — весь код бота
- `requirements.txt` — зависимости
- `.env.example` — пример переменных окружения
- `Dockerfile` — образ для запуска бота
- `docker-compose.yml` — удобный запуск через Docker Compose
- `.dockerignore` — исключения для сборки образа
- `data/bot.db` — SQLite база при Docker-запуске, создаётся автоматически

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

```env
BOT_TOKEN=ваш_токен_бота
DEFAULT_CHAT_ID=-1001234567890
CRON_SCHEDULE=0 18 * * 0
TIMEZONE=Europe/Moscow
```

Описание:

- `BOT_TOKEN` — токен Telegram-бота, обязательный
- `DEFAULT_CHAT_ID` — чат для автоматического weekly poll, необязательный
- `CRON_SCHEDULE` — cron-расписание для автосоздания опроса
- `TIMEZONE` — таймзона планировщика

## Быстрый запуск через Docker на Linux

Нужно, чтобы были установлены Docker и Docker Compose Plugin.

1. Создайте файл `.env`:

```bash
cp .env.example .env
```

2. Откройте `.env` и заполните минимум `BOT_TOKEN`.

Пример:

```env
BOT_TOKEN=1234567890:your_bot_token_here
DEFAULT_CHAT_ID=-1001234567890
CRON_SCHEDULE=0 18 * * 0
TIMEZONE=Europe/Moscow
```

3. Соберите и запустите контейнер:

```bash
docker compose up -d --build
```

4. Посмотреть логи:

```bash
docker compose logs -f
```

5. Остановить бота:

```bash
docker compose down
```

### Как хранится SQLite

В `docker-compose.yml` база хранится в отдельной папке проекта:

```yaml
environment:
  DB_PATH: /data/bot.db

volumes:
  - ./data:/data
```

Это значит:

- данные сохраняются между перезапусками контейнера
- бот запускается одной репликой
- для SQLite это нормальный и практичный вариант

Если раньше вы уже запускали старую версию и Docker создал директорию `bot.db`, удалите её перед новым запуском:

```bash
rm -rf bot.db
mkdir -p data
```

После этого пересоберите и поднимите контейнер заново:

```bash
docker compose down
docker compose up -d --build
```

## Локальный запуск без Docker

Нужен Python 3.11+.

1. Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env`:

```bash
cp .env.example .env
```

4. Заполните `BOT_TOKEN` и при необходимости `DEFAULT_CHAT_ID`.

5. Запустите бота:

```bash
python main.py
```

По умолчанию локально база будет создана как `./bot.db`. При желании можно задать другой путь через переменную окружения `DB_PATH`.

## Как это работает

### Обычный опрос

Команда `/newpoll`:

- создаёт новый poll в таблице `polls`
- сохраняет его как `current_poll_id` для текущего чата
- отправляет новое сообщение через `send_message`
- сохраняет `chat_id` и `message_id` в `poll_messages`

Когда пользователь нажимает `✅ Иду`:

- бот получает `callback_query`
- через toggle добавляет или удаляет пользователя
- пересобирает текст сообщения
- обновляет все связанные сообщения этого poll через `edit_message_text`

### Текущий опрос чата

- для каждого чата хранится `current_poll_id` в `chat_state`
- `/current` отправляет текущее состояние активного опроса в этот чат новым сообщением
- `/closepoll` закрывает текущий poll для этого чата и отвязывает его от чата, не удаляя историю

### Автоматический weekly poll

Планировщик APScheduler по `CRON_SCHEDULE`:

- создаёт новый poll в БД
- отправляет новое сообщение через `send_message`
- сохраняет его в `poll_messages`
- назначает его текущим для `DEFAULT_CHAT_ID`

Важно: для автоматического опроса должен быть заполнен `DEFAULT_CHAT_ID`, иначе задача будет пропущена с предупреждением в логах.

## Формат сообщения

Пример:

```text
Кто идёт на футбол?

Участники:
1. Алексей
2. Иван
3. @username
```

Если список пуст:

```text
Кто идёт на футбол?

Участники:
— пока никого
```

## База данных

Автоматически создаются таблицы:

- `polls(id, created_at, is_closed)`
- `poll_participants(id, poll_id, user_id, username, full_name, created_at)`
- `poll_messages(id, poll_id, chat_id, message_id)`
- `chat_state(chat_id, current_poll_id)`

## Логи

Бот пишет базовые логи в stdout:

- запуск
- создание и закрытие опросов
- добавление и удаление участников
- удаление невалидных сообщений из БД, если Telegram не даёт их редактировать

## Примечания

- Если Telegram вернёт `message is not modified`, это не считается ошибкой
- Если сообщение больше нельзя редактировать или оно не найдено, запись о нём удаляется из БД
- Бот работает как shared-state bot: один и тот же poll можно показывать в нескольких сообщениях, и все они будут синхронно обновляться
- Для `SQLite` не стоит запускать несколько реплик бота одновременно
- Если позже понадобится несколько реплик, лучше перейти с `SQLite` на PostgreSQL
